Like Mother, Like Son

When attempting to model several objects, inheritance can be useful to keep your code DRY. In the real world, living organisms carry around a set fo genetic code. When they have offspring, the child contains nearly identical genetic code. In other words, the child "inherits" traits from the parent. In Object Oriented Programming (OOP), it essentially works the same way. You create a class (the parent), which might contain some methods and attributes. Then you create a child, and tell the child to inherit from the parent. As you might expect, the child looks a lot like the parent it inherits from. In fact, if we create a Child class without telling it to do anything, it will look exactly like the parent. Let's take a closer look at how this works:

There are a few things worth examining here.

First, note that we never define parent_method1 in Child, but we can still use this method. As explained above, it's because we inherited it from the Parent class.

Second, notice the super() on line 19. We told the child to define @type as "child" on 18, and then basically tell it to follow the parent's lead. So when we call it on line 34, it continues with the same functionality (except with a different @type).

Third, we redefine parent_method2 int he child. Even though we never defined parent_method1, we can still use it on the child, and it would exhibit the same functionality as in the parent. If we left parent_method2 alone, it would do the same thing. But, we redefined it, and gave it different functionality.

Fourth, we defined a new method alltogether for the child, child_method, which is unique to the child.

Okay, so everything seems pretty chipper here. Except inheritance can get a bit tricky. If we want to change the Parent class, for instance, the child would be affected by the change.

Ignoring the number lines here (pretend this is the same file, and takes place after the last block of code; I still haven't gotten the hang of using gists), notice that we didn't even touch the Child class, but it is still affected by the updated Parent class. In addition, updating the Parent class affects already existing children. So if we changed parent_method1, calling it on the 'me' instance would yield a different reslult. This isn't the end of the world, but it's still important to watch out.

One problem with this is that Ruby does not have multiple inheritance. In other words, you can only have ONE parent, and ONE parent only. What do you do if you want to inherit methods from other classes or modules?
